TrueMenu

Every town has at least 1 or 2 small family owned pizza restaurants. The thing that these restaurants generally have in common is unfortunately, a terrible website, and a lack of ability to order online. I’m currently building TrueMenu as an exercise that solves part of this problem, and allows even the smallest restaurant to compete with corporate pizza.

API
TrueMenu is designed to function as a JavaScript widget that’s simply dropped within a DOM element. The script with then collect the required components through the site’s api, and dynamically build a menu system specific to the requesting site based on the api key. The requested scripts maintain a reference to the original script and remove it. Essentially the api constructs the widget and then cleans up after itself. (The script to the left is active and will work on any site)
Widget
The TrueMenu widget is designed to integrate seamlessly with the client’s website. With this in mind, I made the choice to avoid any dependencies and write only vanilla JavaScript. The inclusion of a framework like angular would conflict if the client were running their own angular app. This means that the majority of DOM elements are created dynamically by looping over JSON data and assigning corresponding tags and attributes. The JavaScript is minified and contained within a single window variable, $TrueMenu.
Edit-in-Place
All string data with the management system is designed to be editable anywhere it’s displayed to the user. This creates for a more intuitive user experience. Information is displayed within a standard <div> which, on click, is replaced by an imperceptible text input. The data is changed in real time with simple angular data binding.
Flexible Data Structure
Menu items often require a number of options or choices. The system I’ve created so far allows a user to design option sets. For example, a restaurant may carry a menu item, wings. Wings will invariably require additional user choices, ie number, type, sauce. So rather than hard coding choices, the system allows the creation and attachment of option sets. Option sets represent a JavaScript object and can be used globally on many items, or locally on a single item.
"Pizza Builder"
The client widget allows the customization of pizza by end users. This customization is controlled and easily edited within the menu editor. It’s designed to allow the user to intuitively build their desired pizza, and pass that pizza as a JavaScript object to the shopping cart and ultimately the order system.
Receive Realtime Orders via Sockets
A given restaurant has access to an orders screen, linked via sockets. This allows the screen to update with incoming order without the need to refresh the page. Updates can then be pushed back to the customer showing the progress of the order. This lends itself nicely to touchscreen management in the kitchen.
